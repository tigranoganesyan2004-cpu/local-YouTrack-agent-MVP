2. Features that violate Stage 1 rules
src/agent.py — LLM called on deterministic modes by default

LLM_HEAVY_MODES at line 36–49 includes list, count, deadlines, approval_status, overdue, stats, by_customer, by_responsible, with_remarks. For all of these, _should_use_llm() (line 446–449) returns True unconditionally in auto mode — so every count, list, and deadline query runs a two-pass LLM call on top of the already-correct deterministic result. The guide explicitly forbids LLM for counting, filtering, and deadline computation.

src/agent.py — critic pass is active

_run_two_pass_llm() (lines 364–443) always fires the critic pass. The guide says "critic pass remains disabled by default in Stage 1."

src/config.py — reranker is on by default


config.py
Lines 31-31
USE_RERANKER: bool = os.getenv("USE_RERANKER", "true").strip().lower() != "false"
Default is "true". The guide says "must not enable expensive reranking by default."

web/templates/index.html — mode selector is exposed in public UI

Lines 92–98 show a <select> with Auto / Exact / LLM / Deep analysis options. The guide says "Public UI should not show Auto / Exact / LLM / Deep selector."

web/service.py + index.html — streaming box is visible

stream_agent_service() (lines 701–779) is implemented and active. The #streamBox div (lines 114–121) is in the public HTML. The guide says "no raw streaming box in default UI."

web/service.py — deep mode is routed

_route_query() line 74–76 routes web_mode == "deep" to глубоко {query}. Deep mode is a public UI option but should be hidden or disabled by default.

4. Smallest safe first patch
One patch, three lines changed, zero architecture risk:

File: src/config.py Change USE_RERANKER default from "true" to "false".

File: src/agent.py Change _should_use_llm() so it returns False for the deterministic modes (list, count, deadlines, overdue, stats, by_customer, by_responsible, with_remarks, approval_status) in auto mode — i.e., only allow LLM there when analysis_profile == "deep".

File: web/templates/index.html Remove the <option value="deep">Deep analysis</option> and <option value="llm">LLM</option> options from the mode selector (or hide the entire selector), keeping only auto visible.

This patch:

stops the reranker from loading on every search (CPU/RAM win)
stops LLM from running on count/list/deadlines/stats queries (the most common BA queries)
removes the two heaviest public mode options
leaves the critic pass in code but it simply won't be reached for deterministic modes
is fully reversible via .env or by restoring the HTML option
